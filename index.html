I have 40min, so I will not be able to fully introduce Rust or Slint. I'll focus on getting you to a state where you can experiment yourself instead and point out some things I like along the way

The big selling point: It's memory safe. So you can not have dangling pointers ever... That's extensively covered, so I won't :-)

Memory safety leads to less issues with concurrency as the rules introduced to make the language memory safe also happen to prevent data races.

Finally zero cost abstractions: The language is in the same ball-park as C++ with regard to performance and runtime overhead.

Sometimes over-enthusiastic, see RiiR meme. Vut we were the same when we were young. Linus just told us to leave his kernel alone...

processes

Changes are proposed, discussed and implemented in the nightly compiler. They are tested there till there is agreement that the feature works and brings a benefit.

Only then the feature gets declared stable and will then be in the next stable release (which are 6 weeks apart!).

Release testing involves a crater run: A big chunk of the entire language eco-system is built and any new build failure gets investigated.
